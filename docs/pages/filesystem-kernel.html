File System - FreeWorld OS

Location: kernel/fs/
Type: Assembly File System Driver
Purpose: File system operations for loading programs and data
Status: ✅ Basic Implementation Complete

Overview

The FreeWorld File System driver provides basic file operations for the kernel. It supports file open/close, read/write, seek, and directory operations.

Status: Basic file system driver is complete. Total: ~300 lines of code. Currently uses in-memory filesystem; disk-based filesystem to be implemented.

File System Driver (filesystem.asm)

Location: kernel/fs/filesystem.asm (~300 lines)

Basic file system driver with file operations:

Features

  • File open/close
  • File read/write
  • File seek (SEEK_SET, SEEK_CUR, SEEK_END)
  • Directory operations (mkdir, remove)
  • Open file table (64 files max)
  • File existence checking

File Open Modes

  • FS_MODE_READ: 0x01 - Open for reading
  • FS_MODE_WRITE: 0x02 - Open for writing
  • FS_MODE_APPEND: 0x04 - Open for appending
  • FS_MODE_CREATE: 0x08 - Create file if it doesn't exist

Key Functions

Function Description Parameters Returns
init_filesystem Initialize file system EAX = root filesystem base (0 = default) None
fs_open Open a file ESI = file path, EBX = mode flags EAX = file descriptor, or -1 on error
fs_close Close a file EAX = file descriptor EAX = 0 on success, -1 on error
fs_read Read from file EAX = file descriptor, EBX = buffer, ECX = count EAX = bytes read, or -1 on error
fs_write Write to file EAX = file descriptor, EBX = buffer, ECX = count EAX = bytes written, or -1 on error
fs_seek Seek in file EAX = file descriptor, EBX = offset, ECX = whence EAX = new position, or -1 on error
fs_get_size Get file size EAX = file descriptor EAX = file size, or -1 on error
fs_file_exists Check if file exists ESI = file path EAX = 1 if exists, 0 if not
fs_mkdir Create directory ESI = directory path EAX = 0 on success, -1 on error
fs_remove Remove file or directory ESI = path EAX = 0 on success, -1 on error

File Entry Structure

Each open file has a 32-byte structure containing:

  • File descriptor (4 bytes)
  • Inode number (4 bytes)
  • Current position (4 bytes)
  • File size (4 bytes)
  • Open mode (4 bytes)
  • File path pointer (4 bytes)
  • Flags (4 bytes)

Integration

File system is integrated into the kernel via kernel/integration.asm:

; In kernel_subsystems_init:
mov eax, 0  ; Use default base address
call init_filesystem  ; Initialize file system

Status: ✅ Integrated - File system initialized during kernel boot

Usage Examples

Open and Read File

; Open file for reading
mov esi, file_path
mov ebx, FS_MODE_READ
call fs_open
cmp eax, -1
je open_error
; EAX contains file descriptor

; Read from file
mov ebx, buffer
mov ecx, 1024  ; Read 1024 bytes
call fs_read
; EAX contains bytes read

; Close file
call fs_close

Open and Write File

; Open file for writing (create if needed)
mov esi, file_path
mov ebx, FS_MODE_WRITE | FS_MODE_CREATE
call fs_open
cmp eax, -1
je open_error

; Write to file
mov ebx, data_buffer
mov ecx, data_size
call fs_write
; EAX contains bytes written

; Close file
call fs_close

Seek in File

; Seek to beginning
mov ebx, 0
mov ecx, 0  ; SEEK_SET
call fs_seek

; Seek to end
mov ebx, 0
mov ecx, 2  ; SEEK_END
call fs_seek

; Seek relative to current position
mov ebx, 100
mov ecx, 1  ; SEEK_CUR
call fs_seek

Future Enhancements

The current implementation uses an in-memory filesystem. Future enhancements will include:

  • Disk-based file system (FAT32, ext2, or custom format)
  • Directory listing and traversal
  • File permissions
  • Symbolic links
  • File system mounting
  • Journaling for reliability
